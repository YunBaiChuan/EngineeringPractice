## 第一题：1930.长度为3的不同回文子序列 https://leetcode.cn/problems/unique-length-3-palindromic-subsequences/description/?envType=daily-question&envId=2025-11-21

**思路：比较明显的思路，每个$1 <= x <= n - 2$的元素算贡献，不过对于此题来说有去重要求，那么就用$set$去重。然后为了保证扫描一遍就可以完成$ans$的记录，我们可以记录两个数组，一个total和一个$tmp$，这样从前往后扫描的时候，tmp就是遍历到位置的字符数量，与$total$相应位置进行减法之后，就得到当前位置右边的字符数量，最终搞出$ans$即可。注意：在进行$ans$记录的时候，要先把当前位置的$total$给减去，否则会造成答案过大的情况，因为我们本身是不需要当前位置数量的。**

**代码：**
```python
class Solution:
    def countPalindromicSubsequence(self, s: str) -> int:
        total, tmp = [0] * 26, [0] * 26
        for x in s:
            offset = ord(x) - ord('a')
            total[offset] += 1
        
        ans = set()
        for i in range(len(s) - 1):
            offset = ord(s[i]) - ord('a')
            if i == 0:
                tmp[offset] += 1
                continue 
            total[offset] -= 1
            for j in range(26):
                if min(tmp[j], total[j] - tmp[j]):
                    res = chr(ord('a') + j) + s[i] + chr(ord('a') + j)
                    ans.add(res)
            tmp[offset] += 1
            total[offset] += 1
        return len(ans)
```

## 第二题：小苯的计算式 https://www.nowcoder.com/practice/df8990666dea4634a9b746cda5226f00?channelPut=tracker2

**思路：也是比较明显的思路，首先把$n$减去必要两个值，也就是符号和$c$的长度，接着就是比较经典的两数之和优化，我们枚举其中一个数，比如$a$，那么$b$就等于$c - a$，如果满足长度为减少后的n，就加入到$ans$，并且根据$+$的对称性，我们只需要枚举一半的c即可，最终还要判断一下$c$值是否是偶数，他会得到$1 + 1 = 2$这种情况，会多算一次，要把它减去。**
