## 第一题：1262.可被三整除的最大和 https://leetcode.cn/problems/greatest-sum-divisible-by-three/description/?envType=daily-question&envId=2025-11-23

**思路：贪心+分情况讨论，这里我们直接逆向思考去算s的总和并且进行讨论，而不是从0去拼元素来算s，后者的可能性太多了，不好贪心。从总和s来算的话，其实他也只有三种情况：1.s % 3 == 0，此时就是满足条件的s，直接返回即可；2.s % 3 == 1，此时我们有两种可能，一个a1非空是从% 3为1的a1元素中删除一个最小的元素，否则就从% 3为2的a2元素中删除两个最小的元素。3.s % 3 == 2，此时和情况2是一样的，就是删除的顺序不同。为了统一2，3为一套逻辑，我们可以提前if判断一下，如果s % 3 == 2就可以交换a1和a2，最终这几种情况取max即可。**

**代码：**
```python
class Solution:
    def maxSumDivThree(self, nums: List[int]) -> int:
        s = sum(nums)
        if s % 3 == 0:
            return s 
        a1 = sorted(x for x in nums if x % 3 == 1)
        a2 = sorted(x for x in nums if x % 3 == 2)
        if s % 3 == 2:
            a1, a2 = a2, a1 
        ans = s - a1[0] if a1 else 0 
        if len(a2) > 1:
            ans = max(ans, s - a2[0] - a2[1])
        return ans 
```

## 第二题：最小连通代价 https://www.nowcoder.com/practice/479e71a248584e44a63f2d631661b699?channelPut=tracker2

**思路：纯粹的举例子，分类讨论，带点贪心。他分类讨论就是以下这些情况：首先特判一下奇偶数是不是有0，因为后面会有-1操作，造成答案错误。1.a <= 0 and b <= 0，此时我们就要把所有奇偶都给连接成无向完全图，然后对应的奇偶也要连接，这样来达成最小代价。2.a <= 0 and b >= 0，此时把所有奇偶连接成无向完全图，最后奇偶直接连接一条边即可。3.a >= 0 and b <= 0，此时把所有奇偶对应连接即可。4.a >= 0 and b >= 0，此时我们还要讨论下a与b的大小。4.1.如果a >= b，此时我们应该连接成一个双层菊花图，也就是有n - 1条边，并且他们都是奇偶对应的。4.2.如果a < b，此时我们应该分别把奇偶连成一条链，最后奇偶两条链之间再连一条边。这些情况讨论完了，就解决问题了。**

**代码：**
```python
import sys
input = lambda: sys.stdin.readline().strip()

import math
inf = 10 ** 18

def I():
    return input()

def II():
    return int(input())

def MII():
    return map(int, input().split())

def GMI():
    return map(lambda x: int(x) - 1, input().split())

def LI():
    return input().split()

def LII():
    return list(map(int, input().split()))

def LFI():
    return list(map(float, input().split()))

fmax = lambda x, y: x if x > y else y
fmin = lambda x, y: x if x < y else y
isqrt = lambda x: int(math.sqrt(x))

'''
分类讨论，看a和b取值情况，来决定怎么连边可以得到最小连通代价，这个不是kruskal算法，因为他没直接给边权。
'''

def solve():
    n, a, b = LII()
    nums = LII()
    odd = even = 0
    for x in nums:
        if x & 1:
            odd += 1
        else:
            even += 1

    if odd == 0 or even == 0:
        s = fmin((n - 1) * a, n * (n - 1) // 2 * a)
    elif a <= 0 and b <= 0:
        s = (odd * (odd - 1) // 2 + even * (even - 1) // 2) * a + odd * even * b
    elif a <= 0:
        s = (odd * (odd - 1) // 2 + even * (even - 1) // 2) * a + b
    elif b <= 0:
        s = odd * even * b
    elif a >= b:
        s = (n - 1) * b
    else:
        s = (n - 2) * a + b

    print(s)

t = II()
for _ in range(t):
    solve()
```
